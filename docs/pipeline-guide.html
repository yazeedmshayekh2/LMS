Preprocess

Entryrun_preprocessing(config)
Moduleingestion/preprocess/pipeline.py
Runner_stage_preprocess

For each page: geometry → PyMuPDF structure → run pymupdf, pypdf, pdfplumber, structure backends → classify_page → recommend backends → write JSON + previews. Optional Docling markdown for the page range.

Key outputs

  • assets/preprocess/pages/page_NNN.json — full PageProfile
  • assets/preprocess/backends/{name}/page_NNN.txt
  • preprocess_report.json, preprocess_summary.md

Extract

Entryrun_extraction(config)
Moduleingestion/extract/pipeline.py
Depends onPreprocess report (optional)

Loads page profiles via resolve_page_profiles. Per page: layout blocks → region classification → bilingual pairs → tables → vision placeholders → render_raw_page_markdown.

Key outputs

  • raw_markdown/pages/page_NNN.md
  • raw_markdown/document.md
  • extract_report.json

Normalize

Entryrun_normalization(...)
Requiresextract_report.json
Batch IDp001_003 format

build_outline from TOC pages → build_batches → LLM with schema prompts → validate_normalized_markdown vs PyMuPDF reference → write sections + QC.

Frontmatter fields

book_title, grade, content_kind, chunk_id, page_start/end, … — see schema guide in normalize/schema.py.

Readable

Entryrun_readable_export(...)
Modulenormalize/readable_pipeline.py
Requiresnormalized/sections/*.md

Strips YAML frontmatter, runs second LLM pass for reading order only (no new content), writes readable/sections/ and concatenated readable/document.md.

Stage 2 — Extract (deep dive)

flowchart LR
  P[PageProfile] --> LB[layout blocks]
  P --> TB[tables]
  LB --> R[classify_block_region]
  R --> B[bilingual pairs]
  B --> M[render_raw_page_markdown]
  TB --> M
Block regionHow it's chosen
front_matterCover, legal, TOC, preface pages
header / footerTop 11% / bottom 10% of page height
sidebarRight 54% on lesson-like pages
termsBilingual glossary blocks on lessons
mainDefault body content

Stage 3 — Normalize (deep dive)

flowchart TD
  E[load_extract_report] --> O[build_outline]
  O --> B[build_batches]
  B --> L{LLM or dry-run?}
  L -->|LLM| N[normalize_batch_markdown]
  L -->|dry| P[placeholder markdown]
  N --> V[validate_normalized_markdown]
  P --> V
  V --> W[sections + qc_report.json]

Batching rules

  • cover, legal, preface, toc → one page per batch
  • Other types → group consecutive same-type pages up to normalize_batch_size

Stage 4 — Readable (deep dive)

Second LLM pass focused on layout and reading order — no YAML, no invented content. Dry-run writes prompts only and passes through stripped text.

CLI commands

uv run python src/pipeline/run_pipeline.py --max-pages 10 --skip-docling
uv run python src/pipeline/run_pipeline.py --only normalize --provider gemini --batch-size 1
uv run python src/pipeline/run_pipeline.py --from-stage extract --to-stage readable
uv run python src/pipeline/run_pipeline.py --list-stages
